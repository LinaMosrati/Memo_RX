Adressage IPv4

Un PC a besoin de : adresse IP, masque, passerelle par défaut et DNS si on utilise des noms.

ClassePremier octetMasque par défaut
A1 → 126/8
B128 → 191/16
C192 → 223/24
Règle : même réseau + même VLAN = communication directe. Sinon, il faut un routeur ou switch L3.

5

Trunk

Un trunk transporte plusieurs VLAN entre switchs ou entre switch et routeur.

Port trunk 
interface gigabitEthernet 0/1
switchport mode trunk
exit
show interfaces trunk
Limiter les VLAN autorisés
interface gigabitEthernet 0/1
switchport trunk allowed vlan 10,20
exit
Si tu oublies le trunk : les VLAN ne passent pas entre les switchs.

Routage inter-VLAN

Deux VLAN différents ne communiquent pas directement. Il faut un routeur ou un switch niveau 3.

Router-on-a-stick

Sous-interfaces routeur 
interface gigabitEthernet 0/0
no shutdown
exit

interface gigabitEthernet 0/0.10
encapsulation dot1Q 10
ip address 179.16.1.1 255.255.255.0
exit

interface gigabitEthernet 0/0.20
encapsulation dot1Q 20
ip address 179.16.2.1 255.255.255.0
exit

interface gigabitEthernet 0/0.30
encapsulation dot1Q 30
ip address 179.16.3.1 255.255.255.0
exit

Sécurité de base Cisco

Ces commandes reviennent souvent dans les TP : nom de l’équipement, bannière, mot de passe enable, console, VTY, SSH.

Configuration de base sécurisée 
enable
configure terminal
hostname SW1
banner motd # Accès réservé aux utilisateurs autorisés #
enable secret cisco

line console 0
password console123
login
exit

line vty 0 4
password vty123
login
exit

service password-encryption
copy running-config startup-config
Telnet : fonctionne mais n’est pas sécurisé. SSH : est préférable car il chiffre la connexion.
12

Types de routes statiques

Next-hop

On donne l’IP du routeur voisin.

Interface de sortie

On donne l’interface locale de sortie.

Fully specified

On donne interface de sortie + next-hop.

Route flottante

Route de secours avec une AD plus grande.

Modèles IPv4 / IPv6 
! Route statique avec next-hop
ip route RESEAU_DEST MASQUE IP_NEXT_HOP

! Route avec interface de sortie
ip route RESEAU_DEST MASQUE INTERFACE_SORTIE

! Route fully specified
ip route RESEAU_DEST MASQUE INTERFACE_SORTIE IP_NEXT_HOP

! Route par défaut IPv4
ip route 0.0.0.0 0.0.0.0 IP_NEXT_HOP

! Route par défaut IPv6
ipv6 route ::/0 IPV6_NEXT_HOP

! Route flottante IPv4, AD = 5
ip route RESEAU_DEST MASQUE IP_NEXT_HOP 5

! Route flottante IPv6, AD = 5
ipv6 route RESEAU_DEST/PREFIXE IPV6_NEXT_HOP 5

Méthode DVR

  1. Lire l’architecture.
  2. Identifier les réseaux, VLAN, passerelles et trunks.
  3. Configurer les PC.
  4. Configurer VLAN + ports access.
  5. Configurer les trunks.
  6. Configurer routage : statique, router-on-a-stick ou switch L3.
  7. Tester avec ping.
  8. Vérifier avec les commandes show.
Commandes de vérification
show running-config
show ip interface brief
show vlan brief
show interfaces trunk
show ip route
show ipv6 route
show mac address-table
ping
traceroute

Mini QCM DVR

Quelle commande sauvegarde la configuration ?

copy running-config startup-config

Quelle différence entre C et S dans show ip route ?

C = réseau connecté directement. S = route statique ajoutée manuellement.

Quelle commande chiffre les mots de passe visibles ?

service password-encryption

Deux VLAN peuvent-ils communiquer directement ?

Non. Il faut un routeur ou un switch L3.

Quelle commande vérifie les VLAN ?

show vlan brief

Quelle commande active le routage sur switch L3 ?

ip routing

Quelle commande pour Router-on-a-stick ?

encapsulation dot1Q numéro_vlan

Quelle commande désactive la recherche DNS ?

no ip domain-lookup
